How it works

Every colored cell reduces to a score s in 0–1, read on one fixed scale: good at s ≥ 0.66 mixed from s ≥ 0.33 review below. Lower-is-better metrics flip the value first, so the colors always read the same way.

Bounded vs unbounded

Bounded metrics already live in 0–1 and use the thresholds directly. Unbounded metrics (AMC, LCOM1, CBO) have no universal scale, so their values are first normalized against the min–max range of their own column in your report — their colors are relative to sibling types, and a column with no spread stays uncolored. Abstractness and instability are never colored: neither end is universally good.

The n/a dash

A dash (–) means the metric cannot be calculated honestly for that entity — for example, cohesion over method pairs with fewer than two methods. The tool never substitutes a fake zero; hover the dash in the report for the exact reason, also carried in the JSON and CSV reason field.
